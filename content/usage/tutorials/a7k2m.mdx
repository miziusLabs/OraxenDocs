# 🧊 Import a Blockbench model

## Overview

Oraxen uses the standard Minecraft Java resource-pack model format. A Blockbench project must therefore be exported as a JSON model and one or more PNG textures before it can be used by Oraxen.

This tutorial only covers preparing the model files. Configure the exported model as an Oraxen item or furniture item afterwards.

## Export the model

<Steps>

### Use a Minecraft Java project

Create or convert the project to the **Minecraft Java Block/Item** format in Blockbench.

Bedrock models, Java entity models, and `.bbmodel` project files cannot be used directly as Minecraft resource-pack models.

### Create and save the texture

Create or import the texture in Blockbench. Save or export it as a `.png` file.

Use lowercase file names and avoid spaces, for example `wooden_chair.png`.

### Export the JSON model

In Blockbench, export the project as a Minecraft Java block/item model. Save the file as `wooden_chair.json`.

Do not use the `.bbmodel` file as the model inside the resource pack. Keep it separately as the editable Blockbench source file.

### Place the files in Oraxen

For a model named `wooden_chair`, place the exported files in the Oraxen pack directory.

## Set the texture path

Open the exported JSON model and make sure its texture reference points to the correct texture.

```json
{
  "textures": {
    "0": "oraxen:furniture/wooden_chair"
  }
}
```

The namespace is `oraxen`, followed by the texture path below `plugins/Oraxen/pack/textures`. Do not include `textures/` or the `.png` extension in the JSON path.

<Callout type="warning">
  Resource-pack paths are case-sensitive. `Wooden_Chair.png` and `wooden_chair.png` are different files on most servers.
</Callout>

## Test the exported files

Add the model to an Oraxen item configuration. Then run `/oraxen reload all`.

Check the console for model or texture validation errors. Reconnect after the pack has been regenerated so the client receives the new files.

## Troubleshooting

**The texture is missing or purple/black**

Verify that the JSON texture reference matches the texture file exactly, including its path and capitalization.

**The model is a paper item**

Verify that the item uses the same model path as the exported JSON file. The path is relative to `plugins/Oraxen/pack/models` and does not include `.json`.

**The model faces the wrong direction**

Correct the model orientation, pivots, or display settings in Blockbench and export it again. Do not try to fix a wrongly exported model only with furniture translation settings.
