# 🪑 Import Blockbench furniture

This guide imports a Blockbench model as an Oraxen furniture item using `DISPLAY_ENTITY`. It covers the complete path from the exported model to an in-game placement test.

## Before you start

Export the project as a **Minecraft Java item or block model**. Bedrock models, Java entity models, and raw Blockbench project files are not Minecraft resource-pack models and cannot be used directly by Oraxen.

For furniture that needs custom scale or translation, use the documented furniture type `DISPLAY_ENTITY`. Do not use an entity class name as the furniture `type`.

## 1. Prepare the model and texture

Assume the furniture ID is `wooden_chair` and the asset category is `furniture`.

Place the exported files here:

```text
plugins/
└── Oraxen/
    ├── items/
    │   └── furniture.yml
    └── pack/
        ├── models/
        │   └── furniture/
        │       └── wooden_chair.json
        └── textures/
            └── furniture/
                └── wooden_chair.png
```

The model's texture reference must point to the matching pack-relative texture path. With the layout above, use `furniture/wooden_chair` as the path in the model JSON: do not include the `textures/` directory or the `.png` extension.

<Callout type="warning">
  File paths are case-sensitive in resource packs. `Wooden_Chair.png` and `wooden_chair.png` are different files on most servers.
</Callout>

## 2. Create the Oraxen item

Create `plugins/Oraxen/items/furniture.yml` or add this entry to an existing item configuration file:

```yaml
wooden_chair:
  displayname: "<gold>Wooden Chair"
  material: PAPER
  Pack:
    generate_model: false
    model: furniture/wooden_chair
  Mechanics:
    furniture:
      type: DISPLAY_ENTITY
      barrier: true
      hitbox:
        width: 0.8
        height: 1.0
      restricted_rotation: VERY_STRICT
      rotatable: true
      display_entity_properties:
        display_transform: NONE
        scale:
          x: 1
          y: 1
          z: 1
        translation:
          x: 0
          y: 0
          z: 0
        tracking_rotation: FIXED
```

`Pack.model` is the model path relative to `plugins/Oraxen/pack/models` without the `.json` extension.

The `barrier` and `hitbox` are optional but make the furniture interactable and prevent players from walking through it. Adjust their size to fit the model.

## 3. Fix orientation in Blockbench

If the model faces the wrong direction or lies flat when placed, fix its display transform, pivot, or rotation in Blockbench and export it again.

Use `display_entity_properties.scale` and `translation` for size and position adjustments in Oraxen. They are not a substitute for correcting a model that was exported with the wrong orientation.

Example of a smaller chair shifted slightly upward:

```yaml
Mechanics:
  furniture:
    type: DISPLAY_ENTITY
    display_entity_properties:
      scale:
        x: 0.8
        y: 0.8
        z: 0.8
      translation:
        x: 0
        y: 0.1
        z: 0
```

## 4. Regenerate and test

<Steps>

### Reload Oraxen

Run:

```text
/oraxen reload all
```

Watch the console for model or texture validation errors. Fix those before testing in-game.

### Receive the new resource pack

Reconnect or reload the server resource pack so the client receives the regenerated pack.

### Give and place the item

Run:

```text
/oraxen give <player> wooden_chair
```

Place the item and confirm that its model, texture, collision, size, and rotation are correct.

</Steps>

## Troubleshooting

### The item is a paper item or has no custom model

Check all of the following:

- The item ID in `/oraxen give` matches the YAML key exactly.
- `Pack.model` matches the model path below `pack/models`.
- The exported file is a Minecraft Java resource-pack model.
- The model's texture path matches the file below `pack/textures`.
- The client loaded the newly generated resource pack.

### The furniture uses an item frame or cannot be scaled

Set the furniture type to `DISPLAY_ENTITY`. Item-frame furniture has different rendering behavior and does not provide the display-entity properties shown above.

### The model has the right texture but faces the wrong way

Correct its display transform or orientation in Blockbench, then export the model again. Check the model's pivot before trying small translation changes in Oraxen.

### Placement collision is incorrect

Adjust the `hitbox` dimensions, or use multiple `hitboxes` for more complex furniture. The detailed format is documented in [Item mechanics](/configuration/items/mechanics#furniture).
