# 🌐 Resource packs on a proxy

## Overview

This tutorial explains how to use Oraxen resource packs on a Velocity or BungeeCord network without making players download the same pack every time they switch servers.

For PackLayer to skip a resource-pack send, every server involved must generate the same final pack. PackLayer does not merge different packs.

<Callout type="info">
  `multi_version_packs` handles different Minecraft client versions. It does not make separate backend-server packs compatible with each other. Read the [ViaVersion documentation](/compatibility/viaversion) for client-version support.
</Callout>

## Installation

<Steps>

### Install PackLayer on your proxy

Download PackLayer and place it in the `plugins` folder of your Velocity or BungeeCord proxy.

Do not install PackLayer on backend Paper servers.

### Keep the pack sources identical

Every backend server that should share a pack must use the same Oraxen resource-pack source files and the same item, glyph, font, sound, and model configuration that affects pack generation.

Use a deployment method that keeps those files in sync. Copying only `settings.yml` is not enough.

### Generate the pack on every backend

Restart each backend or run:

```text
/oraxen reload all
```

Wait until Oraxen finishes generating and uploading the resource pack before joining or switching servers.

### Use hash-based skipping

Keep PackLayer's default `HASH_ONLY` skip mode:

```yaml
skip-mode: HASH_ONLY
```

This makes PackLayer compare the pack hash before it decides whether to skip a send. It is particularly important when self-hosting, because a pack URL can stay the same after its contents change.

### Test server switching

Join one backend through the proxy and wait until the resource pack has loaded. Then switch to another backend that uses the same pack.

If the pack is sent again, enable `debug: true` in PackLayer and compare the final pack hash generated by both servers.

</Steps>

## Different packs on different servers

If servers need genuinely different packs, PackLayer cannot skip all pack sends.

You can instead send one shared pack from the lobby or entry server and disable pack dispatch on the remaining servers:

```yaml
Pack:
  dispatch:
    send: false
```

Every asset needed on the network must then be included in the pack sent by the entry server.

On Minecraft `1.20.3+`, Oraxen can also use resource-pack layers. Give each separate pack a unique layer name:

```yaml
Pack:
  dispatch:
    layer: "oraxen-survival"
```

Layers allow multiple packs to coexist, but players still have to download each unique pack at least once.

## Troubleshooting

**The pack downloads again after switching servers**

- Confirm that PackLayer is installed on the proxy.
- Compare the final generated pack hash on both servers.
- Make sure both servers finished generation and uploading before the player joined.
- Check PackLayer's server filter and enable `debug: true`.

**Players receive no pack after a restart**

Oraxen needs time to generate and upload the pack. Players who join before this process finishes may not receive the current pack. Wait for the generation and upload messages in the console before allowing players to join.
