# 🔧 Mechanics

## Overview
Mechanics define the behavior and functionality of Oraxen items. They allow you to create simple or complex interactions, or implement custom features such as furniture, blocks, and unique abilities. Mechanics are controlled by the `Mechanics` section in each item config.

## Configuration
Mechanics are configured per item in `.yml` files inside `plugins/Oraxen/items`.

<PluginFileTree
  initialTreeData={[
    {
      id: "plugins",
      name: "plugins",
      children: [
        {
          id: "plugins/Oraxen",
          name: "Oraxen",
          children: [
            {
              id: "plugins/Oraxen/items",
              name: "items",
              children: [
                {
                  id: "plugins/Oraxen/items/weapons.yml",
                  name: "weapons.yml",
                },

                  id: "plugins/Oraxen/items/foods.yml",
                  name: "foods.yml",

/>

```yaml {3-6}
my_backpack:
  material: PAPER
  Mechanics:
    backpack:
      rows: 4
      title: "<red>Backpack"
```

## Enabling/Disabling Mechanics
To improve performance Oraxen allows you to disable / enabled mechanics via the `mechanics.yml` file at `plugins/Oraxen/mechanics.yml`.
You can disable mechanics you don't use or need.

```yaml {2, 5, 8, 11}
itemtype:
  enabled: false

soulbound:
  enabled: true

backpack:
  enabled: true

music_disc:
  enabled: true
```

## Mechanics

<Details>
    <summary id="spear_lunge"><strong>🗡️ `spear_lunge`</strong></summary>
A charged attack mechanic that lets the player lunge forward and hit entities in front of them.<br />

```yaml
my_spear:
  material: IRON_SWORD
  Pack:
    generate_model: false
    model: default/spear_inactive
  Mechanics:
    spear_lunge:
      active_model: default/spear_active
      intermediate_models:
        - default/spear_frame0
        - default/spear_frame1
      smooth_frames: 2
      charge_ticks: 20
      lunge_velocity: 0.8
      max_range: 5.0
      damage: 10.0
      min_damage: 2.0
      knockback: 0.8
      hitbox_radius: 0.5
      min_charge_percent: 0.25
      charge_slowdown: 0.5
      max_hold_ticks: 60
      max_targets: 3
      particles:
        enabled: true
        charge: CRIT
        lunge: SWEEP_ATTACK
        hit: DAMAGE_INDICATOR
      sounds:
        enabled: true
        charge: ITEM_TRIDENT_RIPTIDE_1
        lunge: ENTITY_PLAYER_ATTACK_SWEEP
        hit: ENTITY_PLAYER_ATTACK_STRONG
```

**Options**<br />
`active_model` defines the model shown at full charge.<br />
`intermediate_models` defines optional animation frames during charging.<br />
`smooth_frames` controls how many transition frames are used.<br />
`charge_ticks` defines how long full charge takes.<br />
`lunge_velocity`, `max_range`, `damage`, `min_damage`, `knockback`, `hitbox_radius`, `min_charge_percent`, `charge_slowdown`, `max_hold_ticks`, and `max_targets` control the attack behavior.

**Minecraft version**<br />
`1.21.4+` for model swapping during charge
</Details>

    <summary id="thor"><strong>⚡ `thor`</strong></summary>
Spawns lightning bolts when the item is used.<br />

```yaml
Mechanics:
  thor:
    lightning_bolts_amount: 5
    random_location_variation: 1.5
    delay: 20000
    charges: -1
```

**Options**<br />
`lightning_bolts_amount` defines how many bolts are spawned.<br />
`random_location_variation` defines the random spread in blocks.<br />
`delay` defines the cooldown in milliseconds.<br />
`charges` defines how many uses the item has before being consumed. Use `-1` for unlimited uses.
</Details>

<Details>
    <summary id="knockback_strike"><strong>🥊 `knockback_strike`</strong></summary>
A combo-based mechanic that applies a stronger knockback effect after a configurable number of consecutive hits.<br />

```yaml
Mechanics:
  knockback_strike:
    required_hits: 15
    knockback_horizontal: 2.0
    knockback_vertical: 1.2
    reset_time: 80
    play_sound: true
    sound_type: ENTITY_ENDER_DRAGON_HURT
    sound_volume: 1.5
    sound_pitch: 0.8
    particle:
      type: DUST
      count: 200
      spread: 0.5
```

**Options**<br />
`required_hits` defines how many consecutive hits are needed before the effect triggers.<br />
`knockback_horizontal` and `knockback_vertical` define the knockback force.<br />
`reset_time` defines how long the combo can idle before it resets.<br />
`play_sound`, `sound_type`, `sound_volume`, and `sound_pitch` configure the sound effect.<br />
`particle.type`, `particle.count`, and `particle.spread` configure the particle burst.
</Details>

    <summary id="watering"><strong>💧 `watering`</strong></summary>
Creates a watering can system with linked empty and filled items.<br />

```yaml
empty_watering_can:
  displayname: "<gray>Empty Watering Can"
  material: PAPER
  Mechanics:
    watering:
      filledCanItem: filled_watering_can

    watering:
      emptyCanItem: empty_watering_can
```

**Options**<br />
`filledCanItem` defines which item the empty can becomes when filled.<br />
`emptyCanItem` defines which item the filled can becomes after watering.
</Details>

<Details>
    <summary id="food"><strong>🍖 `food`</strong></summary>
Lets any item behave like food with custom hunger, saturation, replacement items, and potion effects.<br />

```yaml
Mechanics:
  food:
    hunger: 10
    saturation: 10
    replacement:
      oraxen_item: any_oraxen_itemid
    effect_probability: 0.35
    effects:
      hunger:
        amplifier: 1
        duration: 20
        is_ambient: true
        has_particles: true
        has_icon: true
      night_vision:
        duration: 60
```

**Options**<br />
`hunger` and `saturation` define the food values.<br />
`replacement` defines which item replaces the consumed item.<br />
`effect_probability` defines the chance that the listed effects are applied.<br />
`effects` defines potion effects and their settings.

<Callout type="info">
  On `1.20.5+`, prefer the `food` component instead of this legacy mechanic.
</Callout>
</Details>

<Details>
    <summary id="backpack"><strong>🎒 `backpack`</strong></summary>
Turns an item into portable storage.<br />

```yaml
backpack:
  displayname: backpack
  material: PAPER
  unstackable: true
  Mechanics:
    backpack:
      rows: 4
      title: "<red>Backpack"
      open_sound: "entity.shulker.open"
      close_sound: "entity.shulker.close"
```

**Options**<br />
`rows` defines the inventory size.<br />
`title` defines the backpack GUI title.<br />
`open_sound` and `close_sound` define the sounds played when the backpack opens and closes.

<Callout type="warning">
  Use `unstackable: true` for backpack items. Stackable materials can cause duplication issues.
</Callout>
</Details>

<Details>
    <summary id="backpack_cosmetic"><strong>🎽 `backpack_cosmetic`</strong></summary>
Displays a cosmetic backpack on the player using a packet-based armor stand.<br />

```yaml
leather_backpack:
  material: LEATHER_CHESTPLATE
  Pack:
    generate_model: true
    parent_model: item/generated
    textures:
      - leather_backpack
  Mechanics:
    backpack_cosmetic:
      slot: CHEST
      offset:
        x: 0.0
        y: 0.2
        z: -0.2
      scale: 0.8
      view_distance: 48
      hide_while_swimming: true
      hide_while_gliding: true
```

**Options**<br />
`slot` defines where the item must be equipped or stored for the cosmetic to render.<br />
`model` optionally overrides the displayed model.<br />
`offset` defines the rendered position relative to the player.<br />
`scale` defines the rendered size.<br />
`view_distance` defines the maximum visible distance.<br />
`hide_in_spectator`, `small`, and `visible_to_self` control extra render behavior.<br />
`hide_while_swimming` defines whether the backpack gets hidden when swimming (defaults to `true` when not defined).<br />
`hide_while_gliding` defines whether the backpack gets hidden when gliding (defaults to `true` when not defined).

**Global configuration**<br />
```yaml
backpack_cosmetic:
  enabled: true
```
</Details>

<Details>
    <summary id="misc"><strong>🧰 `misc`</strong></summary>
Groups several smaller vanilla-behavior toggles into one mechanic.<br />

```yaml
Mechanics:
  misc:
    breaks_from_cactus: true
    burns_in_fire: true
    burns_in_lava: true
    disable_vanilla_interactions: false
    can_strip_logs: false
    piglins_ignore_when_equipped: false
    compostable: false
    allow_in_vanilla_recipes: true
    prevent_renaming: true
```

**Options**<br />
`breaks_from_cactus`, `burns_in_fire`, and `burns_in_lava` control environmental damage behavior.<br />
`disable_vanilla_interactions` disables normal vanilla interactions.<br />
`can_strip_logs` controls whether the item can strip logs.<br />
`piglins_ignore_when_equipped` makes piglins ignore the player when the item is worn.<br />
`compostable` controls whether the item can be composted.<br />
`allow_in_vanilla_recipes` controls whether vanilla recipes can use the item.<br />
`prevent_renaming` controls whether the item can be renamed.<br />

<Callout type="info">
  On modern versions, several of these behaviors are better handled with components.
</Callout>
</Details>

<Details>
    <summary id="block"><strong>🍄 `block`</strong></summary>
The legacy mushroom stem block mechanic. Its configuration matches `noteblock`, but it is disabled by default and only kept for older setups.<br />

```yaml
Mechanics:
  block:
    custom_variation: 2
    model: my_block
    hardness: 20
    light: 5
    drop:
      silktouch: false
      minimal_type: STONE
      best_tools:
        - PICKAXE
      loots:
        - { oraxen_item: caveblock, probability: 1.0 }
```

**Options**<br />
`custom_variation` defines the unique block variation id.<br />
`model` defines the placed block model.<br />
`hardness` defines break speed.<br />
`light`, `is_falling`, `blast_resistant`, `immovable`, and `can_ignite` control block behavior.<br />
`drop` configures silk touch, tool requirements, and loot tables.<br />
`storage`, `limited_placing`, `blocklocker`, and `clickActions` are also supported like other block mechanics.

**Global configuration**<br />
```yaml
block:
  enabled: true
  tool_types:
    - WOODEN
    - STONE
    - IRON
    - GOLDEN
    - DIAMOND
    - NETHERITE
```

<Callout type="info">
  This is a legacy mechanic. Prefer `noteblock` for new custom blocks.
</Callout>
</Details>

<Details>
    <summary id="noteblock"><strong>🧱 `noteblock`</strong></summary>
Creates custom blocks using note block states. This is the primary block mechanic for most solid custom blocks.<br />

```yaml
Mechanics:
  noteblock:
    custom_variation: 2
    model: my_block
    hardness: 20
    light: 5
    is_falling: false
    blast_resistant: true
    immovable: true
    can_ignite: false
    limited_placing:
      roof: true
      floor: true
      wall: true
      type: ALLOW
      block_types:
        - GRASS_BLOCK
    blocklocker:
      can_protect: true
      protection_type: CONTAINER
    drop:
      silktouch: false
      minimal_type: STONE
      best_tools:
        - PICKAXE
      loots:
        - { oraxen_item: caveblock, probability: 1.0 }
```

**Options**<br />
`custom_variation` defines the unique note block variation id.<br />
`model` defines the placed block model.<br />
`hardness` defines break speed.<br />
`light` defines the emitted light level.<br />
`is_falling`, `blast_resistant`, `immovable`, and `can_ignite` control vanilla-like block behavior.<br />
`drop` configures silk touch, minimum tool type, best tools, and loot tables.<br />
`storage`, `limited_placing`, `blocklocker`, `clickActions`, and `block_sounds` are supported.
</Details>

<Details>
    <summary id="stringblock"><strong>🌿 `stringblock`</strong></summary>
Creates custom tripwire-based blocks, commonly used for plants, flowers, and passable decorations.<br />

```yaml
Mechanics:
  stringblock:
    custom_variation: 9
    model: jasmine_flower
    hardness: 2
    is_tall: false
    light: 10
    blast_resistant: false
    immovable: false
    is_falling: false
    random_place:
      blocks:
        - red_flower
        - blue_flower
    stackable:
      - custom_variation: 41
        model: default/pink_petals_2
    storage:
      type: STORAGE
      rows: 3
      title: "My Storage"
    drop:
      silktouch: false
      loots:
        - { oraxen_item: jasmine_flower, probability: 1.0 }
```

**Options**<br />
`custom_variation` defines the unique tripwire variation id.<br />
`model` defines the placed block model.<br />
`hardness` defines break speed.<br />
`is_tall` makes the block two blocks tall.<br />
`random_place.blocks` lets placement pick randomly from multiple blocks.<br />
`light`, `blast_resistant`, `immovable`, and `is_falling` control block behavior.<br />
`storage`, `blocklocker`, `clickActions`, `stackable`, `sapling`, and `drop` are also supported.

**Global configuration**<br />
```yaml
stringblock:
  enabled: true
  tool_types:
    - WOODEN
    - STONE
    - IRON
    - GOLDEN
    - DIAMOND
    - NETHERITE
```

<Details>
    <summary id="chorusblock"><strong>🪟 `chorusblock`</strong></summary>
Creates custom blocks using chorus plant states. It is especially useful for transparent blocks and simple see-through furniture.<br />

```yaml
Mechanics:
  chorusblock:
    custom_variation: 1
    model: my_chorus_block
    hardness: 5
    light: 12
    is_falling: false
    blast_resistant: true
    immovable: true
    seat:
      height: 0.5
      yaw: 0
    blocklocker:
      can_protect: true
      protection_type: CONTAINER
    storage:
      type: STORAGE
      rows: 3
      title: "My Storage"
    drop:
      silktouch: false
      loots:
        - { oraxen_item: my_chorus_block, probability: 1.0 }
```

**Options**<br />
`custom_variation` defines the unique chorus plant variation id from `1` to `63`.<br />
`model` defines the placed block model.<br />
`hardness`, `light`, `is_falling`, `blast_resistant`, and `immovable` control block behavior.<br />
`seat` allows the block to be sat on.<br />
`drop`, `block_sounds`, `limited_placing`, `blocklocker`, `clickActions`, and `storage` are also supported.
</Details>

<Details>
    <summary id="shaped_block"><strong>🪚 `shaped_block`</strong></summary>
Creates custom stairs, slabs, doors, trapdoors, grates, and bulbs using waxed copper block states.<br />

```yaml
Mechanics:
  shaped_block:
    type: STAIR
    custom_variation: 1
    hardness: 2
    light: 0
    textures:
      bottom: block/oak_door_bottom
      top: block/oak_door_top
    block_sounds:
      break_sound: block.wood.break
      place_sound: block.wood.place
    drop:
      silktouch: false
      fortune: false
      minimal_type: WOODEN
      best_tools:
        - PICKAXE
      loots:
        - { oraxen_item: custom_oak_stairs, probability: 1.0 }
```

**Options**<br />
`type` defines the shaped block type. Supported values are `STAIR`, `SLAB`, `DOOR`, `TRAPDOOR`, `GRATE`, and `BULB`.<br />
`custom_variation` defines the unique variation from `1` to `4` for that shaped block type.<br />
`hardness` and `light` control basic block behavior.<br />
`textures` is used for special cases like doors with top and bottom textures.<br />
`block_sounds` configures place, break, hit, step, and fall sounds.<br />
`drop` configures silk touch, fortune, minimum tool type, best tools, and loot tables.

**Global configuration**<br />
```yaml
shaped_block:
  enabled: true
  convert_vanilla_waxed: true
  handle_world_generation: true
```
</Details>

<Details>
    <summary id="furniture"><strong>🪑 `furniture`</strong></summary>
Creates custom placed furniture using item frames, display entities, armor stands, or other supported furniture render types.<br />

```yaml
table:
  Mechanics:
    furniture:
      type: DISPLAY_ENTITY
      barrier: true
      barriers:
        - origin
        - z: 1
      hitbox:
        - 0,0,0 1,3 # Hitbox with 0,0,0 offset from the furniture with the size 1x3.
        - 1,0,3 0.5,2.5 # Hitbox with 1,0,3 offset from the furniture with the size 0.5x2.5.
        
        # Or the "legacy" single hitbox..
        
        width: 0.4
        height: 0.3
      seat:
        height: 0.5
      restricted_rotation: VERY_STRICT
      rotatable: true
      light: 5
      hardness: 5
      item: different_item_id
      farmland_required: false
      farmblock_required: false
      blocklocker:
        can_protect: true
        protection_type: CONTAINER
      limited_placing:
        floor: true
        wall: false
        roof: false
        type: ALLOW
        block_types:
          - GRASS_BLOCK
      storage:
        type: STORAGE
        rows: 5
        title: "<red>My Storage"
      jukebox:
        active_model: opened
        volume: 1.0
        pitch: 1.0
        permission: "oraxen.jukebox.play"
      evolution:
        delay: 6000
        next_stage: my_plant_stage2
        probability: 0.5
      text_entity:
        text:
          - "<gold>Workshop Table"
        offset: { x: 0.0, y: 1.2, z: 0.0 }
      display_entity_properties:
        display_transform: NONE
        brightness:
          block_light: 15
          sky_light: 0
        scale:
          x: 1
          y: 1
          z: 1
      drop:
        silktouch: false
        loots:
          - { oraxen_item: table, probability: 1.0 }
```

**Options**<br />
`type` defines the furniture render type. Common values are `DISPLAY_ENTITY`, `ITEM_FRAME`, `GLOW_ITEM_FRAME`, and `ARMOR_STAND`.<br />
`barrier`, `barriers`, and `hitbox` define collision and interaction areas.<br />
`seat` allows players to sit on the furniture.<br />
`restricted_rotation` and `rotatable` control placement rotation.<br />
`light`, `hardness`, and `drop` define placed behavior.<br />
`item` defines an alternate display item when placed.<br />
`limited_placing`, `blocklocker`, and `storage` add placement rules, protection, and container behavior.<br />
`jukebox` lets the furniture play music discs and swap to an active model.<br />
`evolution`, `farmland_required`, and `farmblock_required` support plant and farming setups.<br />
`text_entity`, `text_entities`, and `display_entity_properties` configure attached text displays and display entity rendering.<br />
`modelengine_id` is also supported for ModelEngine-based furniture.

<Callout type="info">
  `storage` on furniture requires a barrier or barriers. For modern servers, `DISPLAY_ENTITY` furniture also supports `hitbox` and `display_entity_properties`.
</Callout>
</Details>
